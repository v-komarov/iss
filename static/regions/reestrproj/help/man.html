«GAMMA». Реестр проектов.

Краткое описание



Цель:

Реестр проектов создан с целью автоматизации и контроля за процессами реализации, сдачи в эксплуатацию и бухгалтерского учета законченных строительством объектов сети.

Условия:

В реестре проектов регистрируются все проекты принятые в реализацию техническим блоком.

Проект поступает в реализацию только при условии регистрации в реестре проектов.

В реестре проектов применяются единые форматы именования и шифрования проектов.



Функционал:

Ведение всех реализуемых проектов в одной системе.

График стадий выполнения проектов (контроль сроков реализации и исполнителей).

Привязка и хранение документации проектов в одной системе (финансовые показатели, проектная документация, акты ТМЦ, акты ввода в эксплуатацию, скан-копии документов).

Поиск и формирование отчетов по проектной деятельности.

Ведение и хранение истории взаимодействия по проектам, логи и комментарии.

Автоматическое оповещение участников процесса.

  1. Параметры проекта в реесте.

Далее по мере реализации проекта доступны для работы соответствующие вкладки:



Общие – контрагент (клиент), адрес объекта сети (может быть несколько) – добавляется через «+», связь с системами (шифры проектов)



Показатели, Документы – вкладки для привязки и хранения документов проекта. см. раздел 4.



Стадии – стадии реализации проекта (строительство, оформление документации, проверка, сдача в эксплуатацию и т. П.)

Стадии добавляются из выпадающего списка с помощью кнопки «Установить».

Вся последовательность стадий фиксируется в списке:



Исполнители и даты – содержат всю информацию об уастниках процесса и сроках выполнения задач, см. пункт 4.

Комментарии – здесь хранятся все комментарии и логи действий по проекту.

Дочерние проекты – для поэтапных проектов, когда реализация происходит в несколько этапов, в разных регионах и т. П.

При создании такого проекта он автоматически присваивает порядковый номер и тип подпроекта.
